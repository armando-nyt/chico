chico: index.ts Explained

This page is a guided copy of the prototype library source. The core idea is small: DOM helper functions create real nodes, signals hold values, effects wire signals to exact DOM updates, and cleanup callbacks are attached to nodes so removing DOM also removes reactive subscriptions.

Mental Model

1. Create DOM

html.div(...) and friends call document.createElement. The result is a real browser node.

2. Read signals

When an effect runs, signal reads are tracked. The effect becomes subscribed to the exact signals it used.

3. Update nodes

When a signal changes, subscribers run. Text nodes, props, and styles update directly.

4. Clean on removal

dom.unmount walks the DOM tree, runs cleanup callbacks, and removes the nodes.

Shared Internals

These two values are the little engine room. activeObserver is the currently running reactive observer. cleanupMap stores cleanup callbacks by DOM node without preventing garbage collection. 

let activeObserver = null;

const cleanupMap = new WeakMap();

signal(initialValue)

A signal is a small box around a value. get() returns the current value and records a dependency if an effect or computed value is currently running. set() updates the value and calls subscribers. update() is a convenience wrapper for updating from the previous value. 

export function signal(initialValue) {
  let value = initialValue;
  const subscribers = new Set();

  const api = {
    get() {
      track(api);
      return value;
    },
    set(nextValue) {
      if (Object.is(value, nextValue)) return;
      value = nextValue;
      for (const subscriber of [...subscribers]) subscriber();
    },
    update(fn) {
      api.set(fn(value));
    },
    subscribe(subscriber) {
      subscribers.add(subscriber);
      return () => subscribers.delete(subscriber);
    }
  };

  return api;
}

The important move is track(api). That lets the currently running observer say, "I depend on this signal."

computed(fn)

A computed value is a cached signal-like value derived from other signals. It recomputes lazily: dependencies mark it dirty, but the function only runs again when someone calls get(). 

export function computed(fn) {
  let value;
  let dirty = true;
  const subscribers = new Set();
  const dependencies = new Map();

  const api = {
    get() {
      track(api);
      if (dirty) recompute();
      return value;
    },
    subscribe(subscriber) {
      subscribers.add(subscriber);
      return () => subscribers.delete(subscriber);
    }
  };

  function recompute() {
    for (const unsubscribe of dependencies.values()) unsubscribe();
    dependencies.clear();

    const previousObserver = activeObserver;
    activeObserver = {
      depend(source) {
        if (dependencies.has(source)) return;
        dependencies.set(
          source,
          source.subscribe(() => {
            if (dirty) return;
            dirty = true;
            for (const subscriber of [...subscribers]) subscriber();
          })
        );
      }
    };

    try {
      value = fn();
      dirty = false;
    } finally {
      activeObserver = previousObserver;
    }
  }

  return api;
}

The temporary observer inside recompute() watches whichever signals are read while fn() runs. Before each recompute, old dependencies are unsubscribed so conditional computed values do not keep stale subscriptions.

effect(fn)

An effect is the bridge from reactive values to side effects. In this library, those side effects are DOM updates: set a text node, assign a prop, assign a style, or mount/unmount a conditional region. 

export function effect(fn) {
  const dependencies = new Map();
  let stopped = false;

  const observer = {
    depend(source) {
      if (dependencies.has(source)) return;
      dependencies.set(source, source.subscribe(run));
    }
  };

  function clearDependencies() {
    for (const unsubscribe of dependencies.values()) unsubscribe();
    dependencies.clear();
  }

  function run() {
    if (stopped) return;
    clearDependencies();

    const previousObserver = activeObserver;
    activeObserver = observer;

    try {
      fn();
    } finally {
      activeObserver = previousObserver;
    }
  }

  run();

  return () => {
    stopped = true;
    clearDependencies();
  };
}

Each run clears and re-collects dependencies. That keeps dynamic reads accurate: if an effect reads signal A on one run and signal B on the next, it stops listening to A and starts listening to B.

Text(value)

Text creates a real text node. If the value is reactive, it creates an effect that updates only node.data. No parent, component, or tree re-renders. 

export function Text(value) {
  const node = document.createTextNode(String(read(value)));

  if (isReactive(value)) {
    const stop = effect(() => {
      node.data = String(value.get());
    });
    onCleanup(node, stop);
  }

  return node;
}

The cleanup callback is attached to the text node. If that node is unmounted, the effect is stopped automatically.

When(condition, render)

When is structural reactivity. It does not hide elements with CSS. It inserts real nodes when truthy and removes them when falsey. The region is bounded by comment anchors. 

export function When(condition, render) {
  const start = document.createComment("when:start");
  const end = document.createComment("when:end");
  let mountedNodes = [];

  const fragment = document.createDocumentFragment();
  fragment.append(start, end);

  const stop = effect(() => {
    const shouldMount = Boolean(read(condition));

    if (!shouldMount) {
      clearRegion();
      return;
    }

    if (mountedNodes.length > 0) return;

    mountedNodes = normalizeChildren([render()]);
    end.before(...mountedNodes);
  });

  onCleanup(start, () => {
    stop();
    clearRegion();
  });

  return fragment;

  function clearRegion() {
    for (const node of mountedNodes) unmount(node);
    mountedNodes = [];
  }
}

The start comment owns cleanup for the region-level effect. The children own their own nested cleanup. When the region is removed, dom.unmount recursively stops all bindings below it.

dom.mount, dom.unmount, dom.replace

These functions are intentionally thin wrappers around DOM operations. Their value is cleanup discipline: always use dom.unmount when removing library-managed nodes so bindings are disconnected first. 

export function mount(parent, child) {
  const nodes = normalizeChildren([child]);
  parent.append(...nodes);
  return nodes.length === 1 ? nodes[0] : nodes;
}

export function unmount(child) {
  for (const node of normalizeChildren([child])) {
    cleanupTree(node);
    node.remove();
  }
}

export function replace(parent, oldChild, newChild) {
  const nodes = normalizeChildren([newChild]);
  const firstNode = nodes[0] ?? document.createTextNode("");

  parent.insertBefore(firstNode, oldChild);
  for (const node of nodes.slice(1)) parent.insertBefore(node, oldChild);
  unmount(oldChild);

  return nodes.length === 1 ? firstNode : nodes;
}

export const dom = {
  mount,
  unmount,
  replace
};

html and svg Factories

html and svg are namespace objects backed by a Proxy. Accessing html.div creates a tiny factory for a real element without exporting every tag name at the top level. 

export const html = createElementNamespace((tagName) => document.createElement(tagName));

export const svg = createElementNamespace((tagName) =>
  document.createElementNS("http://www.w3.org/2000/svg", tagName)
);

export function createElement(tagName, ...args) {
  return createElementFrom(document.createElement(tagName), args);
}

function createElementFrom(node, args) {
  const [props, children] = splitElementArgs(args);
  applyProps(node, props);
  node.append(...normalizeChildren(children));
  return node;
}

function createElementNamespace(createNode) {
  const factories = new Map();

  return new Proxy(Object.create(null), {
    get(_, tagName) {
      if (typeof tagName !== "string") return undefined;

      if (!factories.has(tagName)) {
        factories.set(tagName, (...args) => createElementFrom(createNode(tagName), args));
      }

      return factories.get(tagName);
    }
  });
}

This is the element creation layer. html.div({ className: "card" }, html.h1("Kitchen")) remains plain JavaScript, keeps the DOM visible, and avoids noisy document.createElement boilerplate.

Props, Styles, and Events

Props are deliberately DOM-shaped. Event props like onclick become browser event listeners. Style objects assign DOM styles. Reactive prop or style values get effects, so only that one property updates. 

function applyProps(node, props) {
  if (!props) return;

  for (const [name, value] of Object.entries(props)) {
    if (name === "style" && value && typeof value === "object") {
      applyStyle(node, value);
    } else if (name.startsWith("on") && typeof value === "function") {
      bindEvent(node, name.slice(2).toLowerCase(), value);
    } else {
      bindProp(node, name, value);
    }
  }
}

function applyStyle(node, styles) {
  for (const [name, value] of Object.entries(styles)) {
    const assign = (nextValue) => {
      node.style[name] = nextValue == null ? "" : String(nextValue);
    };

    assign(read(value));
    bindReactive(node, value, () => assign(value.get()));
  }
}

function bindEvent(node, eventName, handler) {
  node.addEventListener(eventName, handler);
  onCleanup(node, () => node.removeEventListener(eventName, handler));
}

function bindProp(node, name, value) {
  const assign = (nextValue) => {
    if (nextValue === false || nextValue == null) {
      removeDomValue(node, name);
      return;
    }

    if (name in node) {
      node[name] = nextValue;
    } else {
      node.setAttribute(name, String(nextValue));
    }
  };

  assign(read(value));
  bindReactive(node, value, () => assign(value.get()));
}

function bindReactive(node, value, update) {
  if (!isReactive(value)) return;
  const stop = effect(update);
  onCleanup(node, stop);
}

function removeDomValue(node, name) {
  if (name in node && typeof node[name] === "boolean") {
    node[name] = false;
  } else if (name in node) {
    node[name] = "";
  }

  node.removeAttribute(name);
}

Cleanup

Cleanup is node-owned. Any effect or event listener created for a node registers a callback with onCleanup. Removing a parent walks the entire child tree and runs all callbacks. 

function cleanupTree(node) {
  const callbacks = cleanupMap.get(node);
  if (callbacks) {
    for (const callback of callbacks) callback();
    cleanupMap.delete(node);
  }

  for (const child of [...node.childNodes]) cleanupTree(child);
}

function onCleanup(node, callback) {
  const callbacks = cleanupMap.get(node) ?? [];
  callbacks.push(callback);
  cleanupMap.set(node, callbacks);
}

This is the replacement for a component lifecycle. The DOM node is the ownership boundary; removing it cleans the work attached to it.

Utilities

These helpers keep the public API small. normalizeChildren turns strings, numbers, arrays, nodes, and reactive values into DOM nodes. read unwraps reactive values. track connects reads to the active observer. 

function normalizeChildren(children) {
  const nodes = [];

  for (const child of children.flat(Infinity)) {
    if (child == null || child === false) continue;

    if (child instanceof Node) {
      nodes.push(child);
    } else if (isReactive(child)) {
      nodes.push(Text(child));
    } else {
      nodes.push(document.createTextNode(String(child)));
    }
  }

  return nodes;
}

function read(value) {
  return isReactive(value) ? value.get() : value;
}

function isReactive(value) {
  return value && typeof value.get === "function" && typeof value.subscribe === "function";
}

function track(source) {
  activeObserver?.depend(source);
}

How It Works Together

Here is the chain for a small example: 

const count = signal(0);
const label = computed(() => `Count: ${count.get()}`);

dom.mount(
  document.body,
  html.div(Text(label))
);
  1. signal(0) creates state with a subscriber set.
  2. computed(...) creates a lazy derived value. When read, it tracks that it depends on count.
  3. Text(label) creates a text node and an effect that writes label.get() into that node.
  4. Reading label.get() inside the effect subscribes the effect to the computed value.
  5. Reading count.get() inside the computed subscribes the computed value to the signal.
  6. When count.set(1) runs, the signal marks the computed dirty, then the computed notifies the text effect.
  7. The text effect runs again and assigns only textNode.data. No tree render happens.
  8. If the text node is unmounted, its cleanup callback stops the effect and unsubscribes from the computed value.

That is the whole pattern repeated across the library: create real DOM once, bind exact reactive updates to exact nodes, and attach cleanup to the nodes that own the work.
